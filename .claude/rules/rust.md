# Rust コーディングルール

## エラーハンドリング

- ライブラリ層では `anyhow::Result` を使う
- `unwrap()` / `expect()` は使わない（テストコードを除く）
- エラーには `anyhow::Context` で文脈情報を付与する

## 構造

- モジュールは `mod_name/mod.rs` のディレクトリ構成にする
- pub な型・関数には必要に応じてドキュメントコメントを付ける
- 不要な `pub` は付けない（モジュール内で完結するものは非公開）

## 命名

- 型名: `PascalCase`
- 関数・変数: `snake_case`
- 定数: `SCREAMING_SNAKE_CASE`
- Enum のバリアント: `PascalCase`

## スタイル

- `use` 文はグループ化する: 標準ライブラリ → 外部クレート → 自プロジェクト
- 型の `impl` ブロックは型定義の直後に置く
- `clone()` は必要な場合のみ。可能なら参照を使う
- イテレータチェインを活用し、不要な中間コレクションを避ける

## パフォーマンス

- 不要なアロケーションを避ける（`String` よりも `&str`、`Vec` よりもスライス）
- ホットパスでは `format!` よりも `write!` を検討する

## テスト

- テストは対象モジュール内に `#[cfg(test)] mod tests` として書く
- テスト関数名は `test_<何をテストするか>` にする
